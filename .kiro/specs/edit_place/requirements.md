# 要件ドキュメント

## プロジェクト概要（入力）
全体像（`.kiro/specs/overview.md`）に基づき、「お店の情報を編集・削除する機能」を作成する。ユーザが登録済みデータを最新の状態に保ち、候補の整理に活用できることを目的とする。

## 要件
### 1. 目的
- 1.1 登録済みのお店情報を正確に更新できること
- 1.2 不要になったお店を安全に削除できること
- 1.3 編集・削除の結果を即座に確認できること

### 2. 機能要件
- 2.1 登録済みのお店を編集できる
- 2.2 編集対象の項目は登録時と同等とする
  - 店名
  - 食べログURL
  - 来店ステータス（行った / 行っていない）
  - ジャンル
  - エリア
  - 予算帯
  - メモ
- 2.3 お店を削除できる
- 2.4 編集・削除の完了後に結果が確認できる（一覧または詳細へ遷移）
- 2.5 編集・削除に失敗した場合は理由を明確に表示する

### 3. 入力バリデーション
- 3.1 食べログURLは `tabelog.com` ドメインのURLのみ受け付ける
- 3.2 必須項目（店名・食べログURL・来店ステータス）が未入力の場合は保存できない
- 3.3 食べログURLの重複登録を防止する（同一URLは一意）
- 3.4 削除対象が存在しない場合は適切にエラーを返す

### 4. ユーザー体験
- 4.1 編集は既存の入力内容を初期値として表示する
- 4.2 削除操作は誤操作を防ぐため明示的な確認ステップを持つ
- 4.3 バリデーションエラー時に入力値を保持する

### 5. データ要件（論理）
- 5.1 店データの更新日時を保持する
- 5.2 削除は物理削除を前提とする（論理削除は対象外）

### 6. 非機能要件
- 6.1 編集・削除操作は体感的に速い（保存完了まで待ち時間が長くならない）
- 6.2 失敗時は再試行しやすい導線を提供する

### 7. 受け入れ基準
- 7.1 任意の既存店の情報を更新できる
- 7.2 食べログURLが不正な場合に保存されない
- 7.3 重複URLが登録されない
- 7.4 削除後に一覧から対象店が消える
- 7.5 削除対象が存在しない場合にエラーが表示される

### 対象外
- 編集履歴の保持
- 取り消し（undo）機能
- まとめて編集・削除するバッチ操作
